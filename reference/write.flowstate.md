# Write a `flowstate` as a Flow Cytometry Standard file (FCS 3.1)

A `flowstate` – following [Flow Cytometry
Standard](https://pmc.ncbi.nlm.nih.gov/articles/PMC2892967/) conventions
– is converted to FCS 3.1 and written to disk. The conversion is as
follows:

- `[['keywords']]`, `[['parameters']]`, and `[['spill']]` (if present)
  are converted to character string using a delimiter `"|"` to separate
  keyword-value pairs.

  - this string forms the `TEXT` segment of the FCS file.

- `[['data']]` is written as row-major binary.

  - this forms the `DATA` segment of the FCS file.

- The `HEADER` segment (string) – containing the required offsets – is
  derived from `TEXT` and `DATA`.

The FCS 3.1 compliant file is then written to disk.

*N.B.: During testing, FCS 3.1 files written to disk by write.flowstate
were fully compatible with
[read.flowstate](https://nlaniewski.github.io/flowstate/reference/read.flowstate.md)
and flowCore::read.FCS; issues were only with proprietary software
(i.e., FlowJo). If proprietary software is to be used with FCS 3.1 files
generated by write.flowstate, caution is advised.*

## Usage

``` r
write.flowstate(
  flowstate,
  file.dir = tempdir(),
  fil.suffix = NULL,
  endianness = c("little", "big")
)
```

## Arguments

- flowstate:

  A `flowstate` as returned from
  [read.flowstate](https://nlaniewski.github.io/flowstate/reference/read.flowstate.md).

- file.dir:

  A [file.path](https://rdrr.io/r/base/file.path.html) – default
  [tempdir](https://rdrr.io/r/base/tempfile.html); a file directory for
  saving the output FCS 3.1 file(s).

- fil.suffix:

  Character string – default `NULL`; if defined, the suffix will be
  appended

- endianness:

  Character string – default `"little"`; see
  [endian](https://rdrr.io/r/base/readBin.html).

## Value

- FCS 3.1 file(s) are written to disk and a summary message is
  displayed.

  - the original `$FIL` will be appended with `'flowstateMOD'` as an
    explicit indicator of modification; if `fil.suffix` is defined,
    `'flowstateMOD'` will be further appended with the string value.

- UPDATES BY REFERENCE:

  - `flowstate[['data']]` – non-numeric (logical and/or factor) columns
    are converted to numeric

  - `flowstate[['keywords']]` – non-character (factor) columns are
    converted to character

  - `flowstate[['spill']]` – column names are matched/updated using
    `[['parameters']][['N']]`

- UPDATES BY ASSIGNMENT to environment:

  - `flowstate[['parameters']]` – `[['parameters']]` is updated by
    row-binding derived columns found in `[['data']]`

Invisibly returns `flowstate`.
